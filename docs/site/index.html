local-ydb-toolkit includes a plan-first stdio MCP server for Docker-based local-ydb diagnostics, bootstrap, schema DDL, auth, storage, backup/restore, and upgrades, plus a reusable Codex skill and a GitHub Action for disposable CI tenants.

View repository npm package

Install the MCP server

Use the npm package from any MCP client that can run a local stdio server.

{
  "mcpServers": {
    "local-ydb": {
      "command": "npx",
      "args": ["-y", "--prefer-online", "@astandrik/local-ydb-mcp@latest"],
      "env": {
        "LOCAL_YDB_TOOLKIT_CONFIG": "/path/to/local-ydb.config.json",
        "LOCAL_YDB_MCP_CONTENT_FORMAT": "toon"
      }
    }
  }
}

Mutating tools return planned commands by default. They execute only when the exact tool call includes confirm: true.

What agents can do

Diagnose local-ydb

Run stack inventory, YDB healthcheck, tenant status, node checks, GraphShard checks, auth checks, storage placement, and container logs.

Bootstrap test stacks

Plan and start root-only /local databases or CMS tenant topologies with dynamic nodes when GraphShard or tenant storage is required.

Generate schema DDL

Generate and validate YDB table DDL from structured JSON specs before applying changes through the official YDB JavaScript SDK path.

Manage risky workflows

Use plan-first flows for auth hardening, dump/restore, storage group changes, version upgrades, and stack teardown.

Install the Codex skill

$skill-installer install https://github.com/astandrik/local-ydb-toolkit/tree/main/skills/local-ydb

The skill gives Codex reusable operational rules for Docker local-ydb, CMS tenants, dynamic nodes, auth hardening, storage workflows, and safe verification.

Use local YDB in GitHub Actions

Use astandrik/setup-local-ydb when CI needs a disposable YDB tenant.

- uses: astandrik/setup-local-ydb@v1
  id: ydb
  with:
    version: 26.1.1.6
    tenant: /local/test
