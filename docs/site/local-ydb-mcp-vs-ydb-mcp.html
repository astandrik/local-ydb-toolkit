Comparison

Use local-ydb-mcp for local-ydb operations. Use ydb-mcp for database queries.

The two MCP servers are complementary. @astandrik/local-ydb-mcp operates Docker-based local-ydb environments. The official ydb-platform/ydb-mcp server is for database-level tools against an existing YDB endpoint.

Quick decision table

Need Use Reason
Check Docker containers, local-ydb readiness, logs, storage placement, auth posture, or healthcheck status @astandrik/local-ydb-mcp It knows local and SSH-backed Docker profiles and wraps local-ydb operational checks.
Bootstrap a root /local database or CMS tenant with dynamic nodes @astandrik/local-ydb-mcp It plans and executes local-ydb lifecycle workflows with confirm: true gates.
Run ad hoc SQL, explain queries, inspect paths, or interact with an existing YDB database ydb-platform/ydb-mcp The official server focuses on database-level interaction rather than Docker local-ydb lifecycle.
Generate and validate YDB table DDL from structured JSON specs @astandrik/local-ydb-mcp It provides a read-only generator and an apply path that validates before confirmed execution.

Install local-ydb-mcp

{
  "mcpServers": {
    "local-ydb": {
      "command": "npx",
      "args": ["-y", "--prefer-online", "@astandrik/local-ydb-mcp@latest"]
    }
  }
}

Safety model

@astandrik/local-ydb-mcp is plan-first. Mutating tools return planned commands, risk, rollback notes, and verification steps unless the caller supplies confirm: true. This is useful when AI agents need to inspect a local database stack before changing containers, volumes, auth files, or storage topology.
