GitHub Actions

Start a disposable local YDB tenant in CI.

astandrik/setup-local-ydb boots ghcr.io/ydb-platform/local-ydb, creates a tenant database, waits for readiness, and exports endpoint variables for later workflow steps.

Minimal workflow step

- uses: astandrik/setup-local-ydb@v1
  id: ydb
  with:
    version: 26.1.1.6
    tenant: /local/test

- run: |
    echo "$LOCAL_YDB_ENDPOINT"
    echo "$LOCAL_YDB_DATABASE"

Exported variables

Variable Meaning
LOCAL_YDB_ENDPOINT gRPC endpoint for the disposable local YDB stack.
LOCAL_YDB_DATABASE Tenant database path, such as /local/test.
LOCAL_YDB_MONITORING_URL Local monitoring URL exposed for the job.
LOCAL_YDB_USER and LOCAL_YDB_PASSWORD_FILE Exported only when auth: true is enabled. The raw password value is not printed.

When to use auth

Add auth: true only when the CI job needs native YDB auth behavior. Keep unauthenticated local test stacks simpler when auth is not part of the test contract.

Related MCP workflow

After a stack is running, @astandrik/local-ydb-mcp can inspect health, scheme objects, auth posture, node registration, storage placement, and plan-only schema or lifecycle changes.
