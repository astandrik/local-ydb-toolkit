Tool roundup

Best tools for local YDB workflows with AI coding agents.

Different tools solve different parts of the local YDB workflow. This roundup is intentionally practical: use the smallest tool that matches the job.

Recommended tools

Tool Best for Limit
@astandrik/local-ydb-mcp AI-agent operation of Docker local-ydb stacks: diagnostics, bootstrap, schema DDL, auth, storage, backup/restore, and upgrades. It is an unofficial local stdio MCP server, not a hosted remote service.
ydb-platform/ydb-mcp Database-level YDB interaction such as SQL, query help, directory listing, and path inspection. It is not focused on Docker local-ydb lifecycle management.
astandrik/setup-local-ydb Disposable local YDB tenants in GitHub Actions CI. It is a CI provisioning action, not an interactive MCP diagnostic server.
YDB CLI Direct command-line access for humans and scripted checks. Agents need extra guardrails to avoid unsafe mutations.
ghcr.io/ydb-platform/local-ydb The upstream Docker image used to run local YDB. It provides the runtime image, not agent-facing orchestration or workflow prompts.
skills/local-ydb Codex operational rules for local-ydb topology, auth, storage, verification, and safe runbooks. It guides Codex behavior but does not expose MCP tools by itself.

Best default stack

For AI coding agents working on a repository that needs local YDB, combine the upstream local-ydb Docker image, @astandrik/local-ydb-mcp for operational tooling, and astandrik/setup-local-ydb for CI. Add the official ydb-platform/ydb-mcp when the agent also needs query-level database interaction.

Why plan-first matters

Local database operations can touch containers, volumes, auth files, storage pools, and dumps. A plan-first MCP server lets agents inspect and explain the exact command plan before executing any mutation.
